## 2. Process startup order (run.sh)

1. Export env vars (`ADDON_STATE_ROOT`, `HERMES_HOME`, `HOME`, ‚Ä¶)
2. `mkdir -p` persistent dirs
3. Python inline block ‚Äî reads `/data/options.json`, writes
   `HERMES_HOME/.env` and `HERMES_HOME/config.yaml`
4. `source HERMES_HOME/.env` (set -a / set +a)
5. `unset MESSAGING_CWD` (belt-and-suspenders, see UPGRADE_LOG ¬ßv0.10.4)
6. Start **ttyd** on port 7681 in background
7. Start **server.py** in background
8. Start **hermes dashboard** on 127.0.0.1:9119 in background
9. Run **skills_sync.py** (bundles built-in skills into HERMES_HOME)
10. `exec hermes gateway run` ‚Äî replaces shell; becomes the main process

> **CRITICAL:** Do NOT call `/opt/hermes/docker/entrypoint.sh`.
> It hardcodes `HERMES_HOME=/opt/data`, overwriting our export and
> wiping auth state.  See `UPGRADE_LOG.md ¬ßv0.9.9` and
> `memory/project_hermes_addon_hermes_home_landmine.md`.

## 3. Persistent storage layout (v0.11+)

Map type: **`addon_config:rw`** ‚Äî container `/config` ‚Üî host
`/addon_configs/<slug>_hermes_agent/` (HA 2023.11+ standard, isolated
per add-on, survives uninstall).

```
/config/                          ‚Üê addon_config:rw mount root
‚îú‚îÄ‚îÄ .hermes/                      ‚Üê HERMES_HOME (Hermes state)
‚îÇ   ‚îú‚îÄ‚îÄ .env                      ‚Üê generated by run.sh on every boot
‚îÇ   ‚îú‚îÄ‚îÄ config.yaml               ‚Üê generated by run.sh on every boot
‚îÇ   ‚îú‚îÄ‚îÄ SOUL.md                   ‚Üê agent personality (seeded from upstream)
‚îÇ   ‚îú‚îÄ‚îÄ sessions/                 ‚Üê conversation history
‚îÇ   ‚îú‚îÄ‚îÄ memories/                 ‚Üê agent long-term memory
‚îÇ   ‚îú‚îÄ‚îÄ skills/                   ‚Üê bundled + user skills
‚îÇ   ‚îú‚îÄ‚îÄ hooks/                    ‚Üê event hooks
‚îÇ   ‚îú‚îÄ‚îÄ cron/                     ‚Üê scheduled tasks
‚îÇ   ‚îú‚îÄ‚îÄ logs/                     ‚Üê gateway logs
‚îÇ   ‚îî‚îÄ‚îÄ .addon-runtime            ‚Üê TTYD_CWD sidecar (bash-only, not exported)
‚îú‚îÄ‚îÄ auth/                         ‚Üê AUTH_STORAGE_PATH
‚îÇ   ‚îî‚îÄ‚îÄ session.json              ‚Üê auth bridge state
```

`/data/options.json` ‚Äî Supervisor-managed; read-only input from config UI.
`/data/` itself is ephemeral (cleared on Rebuild).

Host path: `ls /addon_configs/` on HA OS SSH to find the exact `<slug>`.

## 4. URL routing detail

All traffic enters on port **8099** through the HA Ingress proxy.  The
Ingress strips the `/api/hassio_ingress/<token>` prefix and forwards
bare paths to `server.py`.

| Path pattern | Handler |
|---|---|
| `/` | `index.html` (launcher with Dashboard + Terminal buttons) |
| `/styles.css`, `/app.js` | static files |
| `/health` | 200 OK (Ingress health check) |
| `/config-model` | reads `HERMES_HOME/config.yaml ‚Üí model.default` |
| `/panel/**` | reverse proxy ‚Üí `127.0.0.1:9119` (hermes dashboard) |
| `/ttyd/` | reverse proxy ‚Üí `127.0.0.1:7681` (ttyd HTTP) |
| `/ttyd/ws` | WebSocket tunnel ‚Üí ttyd (server.py rewrites origin headers) |
| `/shim/**` | OpenAI shim (provider_shim.py, web-login mode) |

## 7. Common pitfalls checklist (read before every upgrade)

- [ ] Does the new upstream image still have `hermes dashboard`?
      (`hermes dashboard --help` in a test container)
- [ ] Does `hermes gateway run` still work without calling entrypoint.sh?
- [ ] Does the HA WebSocket URL still need the `/api/websocket` patch?
      (grep `api/websocket` in upstream homeassistant.py)
- [ ] Do the Dockerfile ENV paths match run.sh exports?
- [ ] Does `HERMES_HOME` inside the running gateway process point to
      `/config/.hermes`?
      `docker exec <addon> env | grep HERMES_HOME`
- [ ] Does `/config/.hermes/config.yaml` still have `model:` as a dict?
- [ ] Does `curl http://127.0.0.1:8099/config-model` return the real model?
- [ ] Are config.yaml `options:` and `schema:` in sync with
      `translations/en.yaml` and `translations/zh.yaml`?
